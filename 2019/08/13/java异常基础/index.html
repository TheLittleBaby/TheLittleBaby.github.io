  • try、catch、finally三个语句块应注意的问题

    1. try、catch、finally三个语句块均不能单独使用，三者可以组成 try…catch…finally、try…catch、try…finally三种结构，catch语句可以有一个或多个，finally语句最多一个。

    2. try、catch、finally三个代码块中变量的作用域为代码块内部，分别独立而不能相互访问。如果要在三个块中都可以访问，则需要将变量定义到这些块的外面。

    3. 多个catch块时候，最多只会匹配其中一个异常类且只会执行该catch块代码，而不会再执行其它的catch块，且匹配catch语句的顺序为从上到下，也可能所有的catch都没执行。

    4. 先Catch子类异常再Catch父类异常。

    5. 示意图

  • 1.异常
    在原本任务正确的代码/程序中，出现不正常的现象
    中断了程序的进行

    2.异常怎么看
    第一行：异常程序的种类
    最后一行：异常出现的位置
    能看懂哪行看哪行

    3.下列代码可能出现的异常
    3.1 Exception in thread “main” java.util.InputMismatchException
    3.2 Exception in thread “main” java.lang.ArithmeticException: / by zero

    4.完善代码，弥补漏洞
    4.1 if-else:缺点：代码冗余，不一定能堵住所有漏洞
    4.2 处理异常机制* try-catch-finally（异常三连）

    5.try-catch 原理：
    将可能出现的异常的代码放入try中
    然后try中出现异常类型会跟catch后面中的括号进行匹配
    如果匹配成功，执行catch后的代码，捕获异常成功，继续执行后续代码
    如果匹配不成功，try中直接抛出异常，程序中断，后续代码有不会被执行

    6.try-catch处理异常方式
    6.1 自己定义语句：System.out.println(“程序出错”);
    6.2 打印堆栈信息
    ​ System.out.println(e.getMessage()); ​ e.printStackTrace();
    6.3 向上抛出异常，谁调用谁处理* throw e;

  • finally详细讲解

    1.问题“谢谢使用计算器”什么时候不执行
    答
    (1)catch中：throw e;
    (2)catch中没有捕获到对应的异常类型，异常没有捕获成功
    就直接出错，后续代码不执行
    (3)遇到return

    2.若在如何情况下，“谢谢使用计算器”都要执行
    解决办法：try-catch-finally

    3.finally一般放什么内容
    3.1 io流关闭
    3.2 socket关闭
    3.3 数据库资源关闭

    4.在什么情况下，finally不执行
    在try中出现 System.exit(0);
    System.exit(0);终止当前正在运行的java虚拟机

    5.return 和finally的执行顺序
    先执行finally，再return

    6.多重catch解决不同的异常，try中出现的异常，会异常跟catch后的异常比较
    父类异常要写在最后
